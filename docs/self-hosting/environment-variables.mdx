---
title: Environment Variables
description: Every environment variable AnswerLoops uses, with defaults and requirements.
---

Copy `.env.example` to `.env` and fill in the values below.

## Core (required)

| Variable | Description | Example |
|---|---|---|
| `DATABASE_URL` | PostgreSQL connection string | `postgresql://community:community@postgres:5432/community` |
| `NEXTAUTH_URL` | Full URL of your deployment | `http://localhost:3000` |
| `NEXTAUTH_SECRET` | Random 32-byte hex — signs JWT sessions | `openssl rand -hex 32` |
| `ENCRYPTION_KEY` | Random 32-byte hex — encrypts stored API keys | `openssl rand -hex 32` |

## OAuth (at least one required)

| Variable | Description |
|---|---|
| `AUTH_GITHUB_ID` | GitHub OAuth App client ID |
| `AUTH_GITHUB_SECRET` | GitHub OAuth App client secret |
| `AUTH_GOOGLE_ID` | Google OAuth client ID |
| `AUTH_GOOGLE_SECRET` | Google OAuth client secret |

Callback URLs to register:
- GitHub: `{NEXTAUTH_URL}/api/auth/callback/github`
- Google: `{NEXTAUTH_URL}/api/auth/callback/google`

## AI (required for deflection and search)

| Variable | Description |
|---|---|
| `OPENAI_API_KEY` | Platform-default OpenAI key. Per-org keys can be set in Settings → AI Model. |

<Info>
  Per-org AI keys set in the dashboard override `OPENAI_API_KEY` for that org. The platform key is used when no org key is configured.
</Info>

## Discord (optional)

| Variable | Description |
|---|---|
| `DISCORD_TOKEN` | Bot token from Discord Developer Portal → Bot tab |
| `DISCORD_CLIENT_ID` | Application ID from Discord Developer Portal → General Information. Required for 1-click OAuth install. Same value as `DISCORD_APPLICATION_ID`. |
| `DISCORD_APPLICATION_ID` | Application ID — used for slash command registration |
| `DISCORD_CHANNEL_IDS` | Comma-separated channel IDs (env-var fallback if no DB integration set) |
| `DISCORD_GUILD_ID` | Server (guild) ID — limits slash command registration to one server during dev |
| `AUTH_URL` | Public URL of your deployment (e.g. `https://app.example.com`). Required for OAuth redirect URIs. |

<Info>
  Add `{AUTH_URL}/api/discord/callback` to your Discord app's OAuth2 Redirect URIs in the Developer Portal.
</Info>

## Slack (optional)

Slack supports two install paths:

| Path | When to use |
|---|---|
| **1-click OAuth** | Set `SLACK_CLIENT_ID` + `SLACK_CLIENT_SECRET`. Users click "Add to Slack" — no credentials to copy. |
| **Polling (manual)** | Omit OAuth vars. Users paste a bot token directly. No webhook or admin access required. |

| Variable | Description |
|---|---|
| `SLACK_CLIENT_ID` | OAuth app Client ID from `api.slack.com/apps` → Basic Information. Enables 1-click install. |
| `SLACK_CLIENT_SECRET` | OAuth app Client Secret. Required with `SLACK_CLIENT_ID`. |
| `SLACK_SIGNING_SECRET` | App signing secret from Basic Information. Required only if using the Slack Events API webhook (not needed for polling mode). |
| `SLACK_POLL_INTERVAL_SECONDS` | How often the bot polls Slack for new messages. Default: `60`. Minimum recommended: `30`. |

<Info>
  In polling mode the bot calls `conversations.history` on an interval — no public webhook URL required. Ideal for enterprises that cannot grant inbound webhook access.
  
  Add `{AUTH_URL}/api/slack/callback` to your Slack app's OAuth Redirect URLs when using 1-click install.
</Info>

## GitHub App (optional)

| Variable | Description |
|---|---|
| `GITHUB_APP_ID` | GitHub App ID |
| `GITHUB_APP_PRIVATE_KEY` | PEM private key (base64 or raw) |
| `GITHUB_WEBHOOK_SECRET` | Webhook secret for signature verification |

## Push notifications (optional)

| Variable | Description |
|---|---|
| `VAPID_PUBLIC_KEY` | VAPID public key for Web Push |
| `VAPID_PRIVATE_KEY` | VAPID private key |
| `VAPID_EMAIL` | `mailto:you@example.com` — identifies your push endpoint |

Generate VAPID keys:
```bash
npx web-push generate-vapid-keys
```
